# 向导流程

本项目当前采用“开始页 + 独立导入向导弹窗”交互：开始页负责浏览题库、配置、导入与导出；资料导入在单独弹窗中按页面顺序完成资料确认、AI 导题、库内查重、补解析与入库。当前主流程已经移除了“建题库 / 选择录题模式 / 手动录入 / 命名页 / 跨库查重设置页 / 查重选项页 / 解析选项页”等旧环节，页面标题以功能命名为准。

## 开始

- 显示欢迎信息与 API 配置入口。
- 可打开“配置 API…”维护 DeepSeek / Kimi / 千问配置。
- 可打开“常规设定”调整统一并发数、默认题库保存位置等。
- 可通过菜单“导入 -> 从文档文件解析导入”打开独立导入弹窗。
- 可通过菜单“导入 -> 从表格文件直接导入”导入数据库字段表 xlsx。
- 可通过菜单“导出 -> 导出当前层级为 Markdown”导出当前选中层级。
- 可通过菜单导出当前层级或整体数据库表为 xlsx。
- 左侧树展示当前总库层级：
  - `必修三`
  - `第一课`
  - `第一框`
- 右侧表格展示当前总库题目，支持：
  - 视图菜单切换显示列
  - 双击条目编辑
  - 只读浏览题目、选项、答案、解析

## 导入弹窗

- 当前不再出现单独的“查重选项”与“解析选项”页面。
- AI 解析完成后，是否进入“库内查重（结果）”与“AI 生成解析”由程序设定直接决定。

## 确认导入资料

- 页面为左右布局。
- 左侧上方：
  - 选择或拖拽 `docx`
  - 以表格显示当前已选文件、版本、层级
  - 层级要求填写为三级数字格式，例如 `3.2.2`
- 左侧下方：
  - 导入提醒区域
  - 会检查 `Word` 文档中是否存在图片、是否存在表格
- 右侧：
  - 文档预览区域
  - `docx` 转为 PDF 后预览
- 点击“下一步”后自动开始解析。

## AI 解析详情

- 显示解析进度与停止按钮。
- 以表格展示每题三模型返回：
  - DeepSeek
  - Kimi
  - 千问
- 同时显示各模型耗时、轮次与一致性结论。
- 导入抽取采用三模型一致性校验：
  - 每题最多 3 轮
  - 每轮依次由 DeepSeek / Kimi / 千问 各抽取 1 次
  - 同一轮三者结果一致才写入
  - 三轮仍不一致则跳过该题

## 库内查重（结果）

- 直接对“常规设定”指定的当前题库 `.db` 执行查重。
- 无需手工指定查重路径。
- 以只读表格展示相似题对：
  - 当前题目
  - 当前序号
  - 库内来源
  - 所属层级
  - 相似度
- 双击行可打开详情对话框，并列查看当前题目与库内题目的题干、选项与答案。

## AI 生成解析

- 自动读取当前草稿中缺失解析的题目。
- 进入本页后自动开始批量生成，提供进度条与停止生成按钮。
- 生成完成或停止后，自动把解析写回当前草稿数据。
- 参考项来自程序设定与本地参考资料：
  - `reference/resource/` 下的全部 md
  - `reference/mistakes/选择题常见错题归因与答题策略分析.md`

## 导入数据库成功

- 当前批次题目会在进入本页前写入当前题库 `.db`。
- 本页显示本次实际写入题数。
- 关闭弹窗后返回开始页，可继续浏览总库或用开始页菜单导出 Markdown / xlsx。
