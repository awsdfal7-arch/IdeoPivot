# 导出格式

## Markdown

导出为一个 .md 文件。

### 文件头

- 第一行（标题行）：
  - `# 所属部分：{当前层级level_path}  日期：{YYYY年MM月DD日}`
- 标题行后空一行，开始输出题目列表

### 题目列表

- 题目之间空一行
- 每道题的题目块首行格式：
  - `{编号}. {题干第一行}`
  - 注意：编号前有 1 个空格
- 题干中的 `<br>` 按换行处理
- 选项排版规则：
  - 兼容单选与多选两类选项标识：
    - 单选常见：`A.` / `B.` / `C.` / `D.`，以及 `A、` / `B、` / `C、` / `D、`
    - 多选常见：`①` / `②` / `③` / `④`（以及可能出现的 `①、` 等变体）
  - 在上述标识前强制插入换行
  - 目的：把原本“挤在一行”的选项拆成多行
- 不使用 `---` 等分隔线，主要依靠“空行 + 小标题”分段

### 答案与解析区

- 题目列表结束后追加小标题：
  - `## 答案与解析`
- 逐题输出答案与解析，每题格式：
  - 答案行：
    - `**{编号}. {答案}**`（加粗）
    - 答案支持单选与多选：单选如 `A`，多选如 `①②③`
  - 解析：
    - 按原本换行逐行输出
    - 去掉空行
    - 每题解析结束后再空一行

示例（仅展示结构）：

```markdown
# 所属部分：示例题库  日期：2026年03月18日

 1. 题干第一行

A. 选项A
B. 选项B
C. 选项C
D. 选项D

 2. 题干第一行<br>题干第二行

A、选项A
B、选项B
C、选项C
D、选项D

## 答案与解析

**1. A**
解析第一行
解析第二行

**2. ACD**
解析第一行
```

## 当前范围

当前对外导出格式仅包含：

- Markdown
- xlsx

项目内仍存在 `docx -> PDF` 的预览链路，但该能力只用于导入弹窗中的文档预览，不作为菜单导出格式。
