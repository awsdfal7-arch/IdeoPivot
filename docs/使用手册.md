# 思政智题云枢使用手册

## 1. 引言

思政智题云枢立足于思政教学资源建设、题库体系沉淀与教学资料数字化整合的现实需求，致力于搭建一套兼具规范化、智能化与可持续积累能力的题库工作平台。它不仅服务于日常题目录入与资料整理，更着眼于将分散的文档、零散的题目和不断增长的教学经验，汇聚为可管理、可检索、可扩展、可持续演进的高质量题库资产。

项目支持手动新增题目，也支持从 xlsx、docx 等资料中批量导入题目；在导入过程中，可结合 AI 完成题号与题型识别、题目内容解析、题库查重以及题目解析生成。系统提供 Deepseek、Kimi、Qwen 等模型配置入口，并支持参考资料参与解析生成，适用于教材配套题库整理、教学资源沉淀和题目数据标准化管理。在输出环节，项目支持将题库内容导出为 Markdown、PDF、xlsx 等格式，便于后续编辑、分发、归档和再利用。通过贯通录入、解析、查重、生成与导出的完整链路，思政智题云枢力求让题库建设从重复劳动走向体系化沉淀，从资料堆积走向结构化管理，从人工整理走向智能协同。

项目 Github 链接：https://github.com/awsdfal7-arch/IdeoPivot

## 2. 快速开始

### 2.1 登录

首次启动程序后，系统会进入登录页面。本地登录适用于本机直接使用的场景。进入程序后，点击本地登录即可进入主页面，继续进行题库管理、题目录入、资料导入和导出等操作。账号密码登录功能需根据后期应用的实际部署情况实现，本程序当前仅保留该登录方式的结构位置，后续如接入正式账号体系，可具体实现。

### 2.2 配置

初次安装运行程序后，需要进行一系列配置，具体操作方法参考本文档第四章“项目配置”。

## 3. 主页面

### 3.1 侧边栏：题目架构与层级

主页面左侧用于展示题目所属的层级结构。用户可以在此查看和选择题目的归属位置，用于区分不同书目、章节或单元内容。侧边栏中的层级信息包括：书、课题、框题。通过选择不同层级，用户可以筛选并定位对应范围内的题目。

### 3.2 题目信息表

主页面右侧为题目信息表格区域，用于展示题目的主要字段信息。用户可以在表格中查看题干、题型、答案、解析、来源、所属层级等内容，并配合编辑菜单完成新增、修改和导出等操作。用户可以通过视图菜单栏调整表格中显示的列信息，以便根据当前工作需要突出查看不同字段。在题目信息表格区域中，支持通过 `Ctrl + 鼠标滚轮` 调整界面显示比例，便于在大屏查看和细节检查之间快速切换。用户也可以直接拖动表头分隔线手动调整列宽，以获得更适合当前内容长度的显示效果。

## 4. 项目配置

### 4.1 数据位置

程序运行过程中会涉及题库文件、参考资料和其他辅助数据。在正式使用前，要先确认这些数据的保存位置和管理方式。题库位置用于存放当前项目使用的题库数据，正式使用前建议先确认题库文件所在位置，并保持路径稳定，避免因文件移动导致读取失败。参考资料位置用于存放导入和解析过程中需要使用的辅助资料，该位置中的资料可为后续题号识别、内容解析与解析生成提供支持。参考资料通常包括教材内容、解析参考、归因资料以及其他可供 AI 或人工整理时参考的文本材料，正式使用前建议确认参考资料内容完整，并根据需要持续补充和更新。

### 4.2 配置 AI：Deepseek、Kimi、Qwen

程序支持配置多个 AI 提供方（Deepseek、Kimi、Qwen 等），用于题号题型解析、题目内容解析和题目解析生成等场景。用户可在 API 配置界面中填写或维护不同模型提供方的接口信息。完成配置后，必须通过测试功能检查接口是否可正常连接。以下为各项详细配置说明：

- **程序配置**：用于控制导入和解析流程中的若干默认行为。
- **序号题型解析配置**：控制题号与题型识别相关的处理方式，影响 docx 导入流程中对题目编号、题型归类等信息的解析结果。
- **题目内容解析配置**：控制题目正文、选项、答案等内容的提取与整理方式，主要影响导入时题目结构化结果的生成质量。
- **题目解析生成配置**：控制 AI 生成题目解析时的相关行为，如是否生成解析、参考资料如何参与生成、生成结果的整体风格等。
- **并发 API 解析**：程序支持并发 API 调用，用于提升导入和解析效率。实际使用时可根据接口稳定性、调用限制和机器性能合理调整并发数量。
- **统一测试**：在配置多个 AI 提供方后，建议使用统一测试功能检查接口连通性和配置完整性，以便在正式导入前提前发现异常。

### 4.3 费用日志

费用日志用于记录接口调用过程中的费用和时间信息，便于追踪使用情况和后续统计。费用日志的记录节点包括：程序启动时、接口测试时、导入结束时。费用日志会保存在程序使用的配置或日志目录中，便于后续查看和留档。用户可根据需要清理历史费用日志，以保持记录简洁，或在新的统计周期开始前重置日志内容。

## 5. 题目录入

### 5.1 手动新增题目

用户可以通过编辑菜单栏中的新增题目功能，手动录入新的题目数据。该方式适用于单题录入、人工补录和局部修订等场景。录入时需填写以下信息：

- **题目内容**：题干的正文内容，确保表达完整、清晰。
- **题目类型**：程序支持单选、多选、可转多选三种类型，用户应根据题目实际特征选择合适的题型。
- **选项与答案**：对于选择题，需补充完整的选项内容，并填写或确认正确答案。
- **解析内容**：如已有现成解析，可在新增题目时一并填写，便于后续直接使用或导出。
- **所属层级**：题目录入时需要指定所属层级（书、课题、框题），以便后续在树形结构中正确归类和管理。
- **教材版本**：如题目与具体教材版本相关，建议在录入时明确填写，方便后续筛选和使用。
- **题目来源**：可记录题目的资料来源、整理来源或其他备注信息，便于后续追踪。

### 5.2 导入题目

除手动新增外，程序也支持通过外部文件批量导入题目。对于结构较规范的题目表格，可通过 xlsx 文件进行导入，导入前建议检查列结构是否完整、字段表达是否统一。对于以文档形式整理的题目资料，可通过 docx 导入，相关细节将在第 6 章单独说明。

## 6. docx 导入详解

### 6.1 单独导入 / 并发导入

程序支持单独导入单个文档，也支持根据实际需求进行并发导入。处理多个文档时，建议结合机器性能和 API 并发配置合理安排任务数量。

### 6.2 确认导入资料

在正式开始解析前，用户需要先确认导入资料是否正确，确保后续识别结果具备良好的基础。

- **明确层级**：导入前应明确当前资料对应的书目、章节或单元层级，以便导入完成后题目能正确归入目标位置。
- **表格处理**：若导入文档中包含表格内容，应在导入前确认表格中的题目信息是否完整、顺序是否清晰，避免因格式复杂影响解析质量。
- **图片处理**：若文档中包含图片，应提前确认图片内容是否为题目有效信息，以及图片与正文之间的对应关系是否明确。

### 6.3 题号与题型解析

程序会对导入文档中的题号和题型进行识别与整理，以建立后续题目拆分与结构化提取的基础。

### 6.4 题目内容解析

在题号和题型识别完成后，程序继续对题目正文、选项、答案和相关字段进行内容解析，形成结构化题目数据。

### 6.5 题库查重

解析出的题目在正式入库前，可以与现有题库进行查重，用于识别重复题目、相近题目或疑似重复内容，减少重复录入。

### 6.6 AI 生成解析

在需要补充题目解析时，程序可调用 AI 生成解析内容。该过程可结合参考资料、归因资料和其他辅助文本进行综合生成。系统可参考“选择题常见错题归因.md”中的内容，辅助生成更具针对性的题目解析。除固定参考资料外，也可根据实际需要引入其他资料作为解析生成时的参考依据。注意整理参考内容时应关注输入内容的长度限制（100k），若资料过长，建议先进行筛选和压缩，以免超过可用字符范围。

### 6.7 完成导入

完成导入后，题目将进入题库，用户可返回主页面继续查看、编辑、导出或进行后续整理。

## 7. 导出与生成

### 7.1 导出 Markdown

程序支持将题目内容导出为 Markdown 格式，便于继续编辑、整理或用于其他系统。

### 7.2 导出 PDF

程序支持将题目内容导出为 PDF 格式，便于打印、分发和归档。

### 7.3 导出 xlsx

程序支持将题目内容导出为 xlsx 表格格式，便于数据整理、进一步筛选和批量处理。
