# 数据结构与题库格式

当前项目的题库浏览、编辑与查重主存储为 `SQLite` 总库；导出与兼容流程仍支持 `xlsx`。本文档描述当前数据结构与显示/导入规则。

## 总库 SQLite

- 当前总库文件：
  - `converted_db/思政题库.db`
- 主表：
  - `questions`

### 字段

- `id`：题目唯一标识
- `stem`：题干
- `option_1`：选项 1
- `option_2`：选项 2
- `option_3`：选项 3
- `option_4`：选项 4
- `answer`：答案
- `analysis`：解析
- `question_type`：题型，当前使用：
  - `单选`
  - `多选`
  - `可转多选`
- `textbook_version`：教材版本
- `source_filename`：来源文件名
- `level_path`：所属层级
- `difficulty_score`：难度评级
- `knowledge_points`：考察知识点
- `abilities`：考察能力
- `created_at`：入库时间
- `updated_at`：最后修改时间

## 兼容导出 Excel

- 导出与兼容流程仍可使用 `xlsx`
- 工作表名：
  - `questions`
- 标准表头：
  - `编号`
  - `题目`
  - `选项`
  - `答案`
  - `解析`

字段命名映射：

- `编号` -> `id`
- `题目` -> `stem`
- `选项` -> `options`
- `答案` -> `answer`
- `解析` -> `analysis`

## 题型与显示规则

资料中常见两类选项标识：

- 单选：`A / B / C / D`
- 多选：`① / ② / ③ / ④`

当前开始页表格显示规则：

- `单选`
  - 选项列显示为 `A. / B. / C. / D.`
  - 答案列保持单选答案原样显示
- `多选` / `可转多选`
  - 选项列显示为 `①. / ②. / ③. / ④.`
  - 答案列中 `1,2,3,4` 会显示为 `①②③④`

## level_path 规则

- 当前项目中的层级使用三级编号形式
- 例如：
  - `3.1.1`
  - `3.4.2`
  - `3.6.3`
- 开始页左侧会按其解析为：
  - `必修三`
  - `第一课`
  - `第一框`

## 导入支持

- 支持导入：
  - `*.txt`
  - `*.docx`
- 导入页附带：
  - `Word` 图片检查
  - `Word` 表格检查
  - `txt` 文本预览
  - `docx` PDF 预览

## 校验建议

- 题目非空
- 选项至少 2 项
- 答案与题型匹配
- `level_path` 非空
- 对于导入文档，建议提前确认题号与题干边界清晰
