# 项目架构

## 当前分层

- `app`
  - 负责应用启动与引导。
  - 当前 `main.py` 进入 `sj_generator.app.bootstrap`，再转到桌面端启动入口。
- `presentation`
  - 负责桌面向导、窗口和页面注册。
  - 当前主入口已通过 `presentation/qt` 装配主向导与导入向导。
- `application`
  - 负责用例编排、会话状态、程序设定与跨模块业务入口。
  - 当前导入流程中的文档读取、题号解析、题目解析、草稿入库入口已收口到 `application/importing`。
- `domain`
  - 负责题目等核心领域实体与领域数据结构。
- `infrastructure`
  - 负责 `docx` 读取、LLM 调用、SQLite / xlsx 持久化、Markdown / PDF 导出等底层实现。
- `ui`
  - 负责 Qt 页面、对话框、表格渲染、线程、信号与交互适配。
  - 当前部分页面实现仍在该层，属于迁移中的保留区域。
- `shared`
  - 负责路径等共享能力。

## 当前依赖方向

- 推荐依赖方向为：`presentation / ui -> application -> domain / infrastructure`。
- `app` 只负责启动与拼装，不承载页面业务。
- `ui` 负责界面交互，不直接承担导入主流程编排。

## 导入流程边界

- 导入向导页面仍由 Qt 页面驱动，线程和信号处理保留在 `ui`。
- `ui/import_workers.py` 当前继续承接线程 worker，但 worker 内部已改为调用 `application.importing`。
- `ui/import_db_service.py` 当前负责费用冻结、提示框与状态写回，真正的草稿入库入口已改为 `application.importing.commit_questions_to_db`。
- `application/importing/services.py` 当前集中提供：
  - `load_docx_sources`
  - `resolve_question_refs_for_sources`
  - `import_questions_for_sources`
  - `commit_questions_to_db`

## 当前迁移状态

- 旧的 `ai / io` 目录已不再作为项目主结构说明。
- 当前主骨架已经切换为 `app / presentation / application / domain / infrastructure / ui / shared`。
- `presentation` 与 `ui` 目前仍并存：
  - `presentation` 更偏窗口与页面装配入口。
  - `ui` 更偏具体页面、组件、线程和交互细节。
- 后续如果继续优化架构，应优先把页面装配与业务边界继续从 `ui` 向 `presentation` 和 `application` 收口，而不是先改底层实现。
